# 검수(검증) 알고리즘 강화 — T1 같은 케이스가 '확인필요'로만 끝나던 문제

## 무엇이 문제였나
1. **기준이 너무 느슨**: '척추축과 나란함' 판정 기준이 40°라,
   종판이 45° 기울어도 통과했습니다. 정상이면 종판은 척추축과 거의 **수직(90°)**
   이어야 하는데, 그 기준으로 보지 않았습니다.
2. **신호 2개 이상이어야 보정**: T1처럼 이웃(C7)도 같이 틀리면 '이웃과 급변'이
   안 걸리고, 상·하가 나란히 틀리면 'wedge 모순'도 안 걸립니다.
   결국 가장 확실한 신호 1개만 남아 → 확인필요 표시만 되고 교정은 안 됐습니다.

## 수정
1. **수직성 기준으로 재정의 + 가중치**
   - 종판↔척추축 각이 45° 미만 → 심각(3점), 60° 미만 → 의심(2점), 70° 미만 → 경미(1점)
   - 즉 **이 신호 하나만으로도 보정이 발동**합니다.
   - 사유 문구에 실제 각도를 표시: 예) `종판이 척추축과 45° (수직에서 크게 벗어남)`
2. **보정 방식 개선**: 단순 y축 폴백 대신 **국소 척추 진행축에 수직인 축**으로 재계산
   (해부학적으로 타당). 이미지가 기울어져 있어도 동작.
3. **교차 보정**: v1(4코너)이 못 고치는 모양이면 v2(그룹 피팅)로 한 번 더 시도.
4. **개선 검증**: 보정 결과가 실제로 더 나은 경우에만 적용하고,
   아니면 'review'로 남겨 사람이 보게 함(엉뚱한 값 강요 방지).
